Aside from the content, there are quite a few features that have been implemented:

  • Contact Form for customer feedback
  • Dynamic webpage
  • User login
  • Database API

In which the following section will elaborate more about each of the feature's implementation

Features

Contact Form

This feature allows user to submit their comments or feedbacks to the project development team.

As you can see, the form has been included with the sender's information as well as their comments and rating for the product solution. After clicking the submit button, their inputs will be saved and display in the table underneath the form as a illustration that their form has successfully been updated in the system. Not just in display, but the data will also be send back to the server database for the development team to access and review.

And not only that, the form has been implemented with the input validation function, in which it checks whether there is any problem with the user's filled in details before final updates. There will be a display to notify the user whether there is any issue.

Dynamic View

As you explore the website, you will see sometimes the webpages have dynamic content, such as in the solution webpage, there is a tab layout that can drop down upon click

Contents like those are being implemented by either directly adding JavaScript into the HTML file, or writting the function in a seperate JS file and link it in HTML by declaring

<script src="[JS file name]"> </script>

Login

This feature can be viewed in the Eden Market page, as at the current version, the user credential is only required to access the database that contains the products that Edem sell to users via their software solution.

To ensure the security for user when inputting the login details, the password will be encoded using SHA256 hashing method. The hashed password will be used for both analysing the user input and being saved inside the database.


And not only that, the form has been implemented with the input validation function, in which it checks whether there is any problem with the user's filled in details before final updates. There will be a display to notify the user whether there is any issue.

The current login credentials is a copy of the username/password pair in task 10.2C. You can enter drummerboy68/starrynight as the credential to access the database

Database API

This feature can be accessed after the user successfully log in the webpage.

The database is dynamically generated by using JavaScript to fetch a local JSON file that contain the data of the product name, category, image name and calories index.

By default, the webpage will display all the database's items, however user can filter the items either by selecting a specific category in the drop down selection or type in search bar
